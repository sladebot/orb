Model Tier Selection

Each message is scored 0–100. The score determines which model tier is used. If the preferred tier's provider is unavailable, the selector walks up to the next available tier.

Score range Tier Anthropic model OpenAI model Drivers (+points)
0–30 LOCAL_SMALL — — Ollama qwen3.5:9b · low complexity hint (−15)
31–55 LOCAL_MEDIUM — — base=50 · default for most messages
56–70 LOCAL_LARGE — — Ollama qwen3.5:27b · large payload (+10)
71–78 CLOUD_LITE haiku-4-5 gpt-4o-mini feedback msg (+20) · deep chain (+15)
79–90 CLOUD_FAST claude-sonnet-4-6 gpt-4o high complexity hint (+25) · retry escalation (+10×n)
91–100 CLOUD_STRONG claude-opus-4-6 o3 max score · extreme depth + high hint + feedback

Key Components

LLMAgent
orb/agent/llm_agent.py
  • Async event loop, processes one Message at a time
  • Calls LLM with tool schema; handles tool_use / tool_result pairs
  • Up to 3 nudges if LLM returns text-only (no tool call)
  • Filesystem tools: read_file, write_file, list_dir, run_command
  • Fallback chain: preferred tier → CLOUD_LITE → CLOUD_FAST → CLOUD_STRONG
MessageBus
orb/messaging/bus.py
  • Routes messages between named AgentChannels
  • Validates graph edge exists before delivery
  • HopCounter: rejects if depth > 10
  • BudgetTracker: global cap (default 200 messages)
  • CooldownTracker: max 5 sends on same edge per chain
  • Emits "routed" events to dashboard listeners
Orchestrator
orb/orchestrator/orchestrator.py
  • Starts all agents concurrently, injects initial task
  • Consensus: when any agent completes, broadcasts COMPLETE to others
  • Synthesis flow: coordinator waits for all workers, then synthesizes
  • 600s timeout, then forcibly stops all agents
  • Cleans up sandbox after run
ConversationHistory
orb/agent/conversation.py
  • Per-agent message history, max 20 messages
  • Smart trim: preserves tool_use ↔ tool_result pairs
  • Forward scan finds clean plain-user start point
  • Fallback: backward scan preserves most recent valid slice
  • Prevents orphaned tool_results → Anthropic 400 errors
ProviderRegistry
orb/llm/registry.py
  • Discovers available providers at startup
  • Checks env vars (ANTHROPIC_API_KEY, OPENAI_API_KEY)
  • Pings Ollama at localhost:11434 for liveness
  • Returns dict of instantiated LLMClient objects
  • --local-only / --cloud-only flags filter providers
DashboardBridge
web/bridge.py
  • Listens to MessageBus "routed" events
  • Updates DashboardState (agents, messages, budget)
  • Broadcasts JSON events over WebSocket to all clients
  • Events: message · agent_status · agent_stats · complete · stats
  • Complexity score included in agent_stats events
ModelSelector
orb/llm/model_selector.py
  • Scores each message 0–100 based on complexity signals
  • base_complexity per agent (coordinator=50, tester=25, reviewer=65)
  • Exposes last_score for dashboard/TUI display
  • Self-escalates on retries (+10 per retry)
  • select_with_available() skips unavailable provider tiers
Sandbox
orb/sandbox/sandbox.py
  • Isolated filesystem root per orchestrator run
  • Path traversal prevention (all paths validated)
  • Shared across all agents in the same run
  • Async run_command with stdout/stderr capture
  • Cleaned up after each run completes
